# 🔧 설치 방식 비교: 어떤 것을 선택할까?

---

## 📊 3가지 설치 방식 비교표

| 기능 | MCP 서버 | Slash Command | 플러그인 |
|------|---------|---------------|---------|
| **설치 난이도** | 어려움 | 중간 | 쉬움 ⭐ |
| **사용 난이도** | 어려움 | 쉬움 | 매우 쉬움 ⭐ |
| **슬래시 커맨드** | ❌ | ⭐ ✅ | ⭐ ✅ |
| **설정 UI** | ❌ | ❌ | ✅ ⭐ |
| **상태 바 표시** | ❌ | ❌ | ✅ ⭐ |
| **다른 도구 지원** | ✅ | ✅ | Claude Code만 |
| **배포 복잡도** | 낮음 | 중간 | 높음 |

---

## 🎯 어떤 것을 선택할까?

### ✅ Claude Code 플러그인 선택!
당신이 **Claude Code만 사용**하고, **가장 편리한 경험**을 원한다면:

```
✨ 권장: /plugin install github:yourusername/harness-builder
```

**이유:**
- 설치가 가장 간단함
- 슬래시 커맨드 지원
- 설정 UI 제공
- 상태 바 표시
- 가장 통합적인 경험

---

### ✅ Slash Command 선택
**빠른 실행**이 가장 중요하고, 복잡한 설정을 원하지 않는다면:

```
⚡ 권장: npm install --save-dev claude-slash-harness
```

**이유:**
- 설치가 비교적 간단
- 슬래시 커맨드로 빠른 실행
- `/harness-init` 형태로 직관적

---

### ✅ MCP 서버 선택
**다양한 도구**와 **유연한 통합**을 원한다면:

```
🔌 권장: npx harness-builder-mcp
```

**이유:**
- 다양한 MCP 클라이언트 지원 (Claude Desktop, 다른 LLM도구 등)
- 더 유연한 구성
- 팀 전체가 공유 가능

## 💡 조합 사용하기

### 최고의 경험 = 플러그인 + MCP 서버!

```
Claude Code에서:
1. 플러그인 설치
2. MCP 서버도 설정

그러면:
- ⚡ 슬래시 커맨드로 빠른 실행
- 🔧 MCP로 더 복잡한 작업
- ⚙️ 설정 UI 사용 가능
- 🌐 다른 도구와도 연동 가능
```

### 설정 예시

```json
{
  "mcpServers": {
    "harness-builder": {
      "command": "npx",
      "args": ["-y", "harness-builder-mcp"]
    }
  }
}
```

## 🎯 추천 설정 (최적)

### 1단계: Claude Code 플러그인 설치
```bash
/plugin install github:yourusername/harness-plugin-complete
```

### 2단계: MCP 서버도 추가 (선택)
설정 파일에 추가:
```json
{
  "mcpServers": {
    "harness-builder-mcp": {
      "command": "npx",
      "args": ["-y", "harness-builder-mcp"]
    }
  }
}
```

### 3단계: 사용!
```
슬래시 커맨드:
/harness-init my-project

또는 MCP 도구:
도구 선택 → init_harness → 실행
```

```
특징:
- 완벽한 통합
- 설정 UI
- 상태 표시

```

## ✅ 최종 권장

### 🏆 가장 좋은 선택: **Claude Code 플러그인**

