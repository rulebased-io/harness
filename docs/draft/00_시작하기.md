## ⚡ 5분 안내

### 빠른 시작 (기존 플러그인 사용)

1. **Claude Code 설정 파일 열기:**
   ```bash
   # macOS/Linux
   nano ~/.config/Claude/claude_desktop_config.json
   
   # Windows
   notepad %APPDATA%\Claude\claude_desktop_config.json
   ```

2. **다음 추가:**
   ```json
   {
     "mcpServers": {
       "harness-builder": {
         "command": "npx",
         "args": ["-y", "harness-builder-mcp"]
       }
     }
   }
   ```

3. **Claude Code 재시작**

4. **사용 시작:**
   ```
   Claude Code에서:
   "Jest 기반의 TypeScript 하네스를 'my-test'로 초기화해줄래?"
   ```

## 📦 플러그인이 제공하는 기능

### 5가지 핵심 도구

| 도구명 | 설명 | 예제 |
|--------|------|------|
| **init_harness** | 테스트 프로젝트 초기화 | Jest/Mocha/Vitest 설정 생성 |
| **generate_test_case** | 개별 테스트 케이스 생성 | 로그인 테스트, API 테스트 |
| **generate_integration_suite** | 여러 테스트를 하나로 통합 | 사용자 CRUD 전체 테스트 |
| **generate_performance_harness** | 성능 테스트 생성 | API 응답 시간 측정 |
| **get_harness_config** | 현재 설정 조회 | 설정 상태 확인 |

## 💡 실제 사용 예제

### 예제 1: API 테스트 자동 생성
```
Claude Code:
"Express API의 /api/users 엔드포인트를 테스트하는

Claude Code:

통합 하네스를 만들어줄래?"

결과:
✅ 마이그레이션 테스트
✅ Create, Read, Update, Delete 테스트
✅ beforeAll/afterAll 자동 설정

```

### 예제 3: 성능 테스트
```
Claude Code:
"API 응답이 100ms 이내인지 확인하는 
성능 테스트를 만들어줄래?"

결과:
✅ performance.now() 측정 코드 자동 생성
✅ 임계값 설정 가능 (기본 100ms)
✅ 모든 API에 대해 생성 가능
```

## 🔧 플러그인 직접 개발하기

### 준비물
- Node.js 18+ 설치
- TypeScript 기본 이해
- npm 사용 경험

### 개발 단계

1. **클론 또는 코드 복사**
   ```bash
   mkdir harness-mcp-plugin
   cd harness-mcp-plugin
   # 폴더의 파일들을 복사
   ```

2. **의존성 설치**
   ```bash
   npm install
   ```

3. **빌드**
   ```bash
   npm run build
   ```

4. **로컬 테스트**
   ```bash
   npm run mcp:inspect
   ```

5. **Claude Code에 연결**
   ```json
   {
     "mcpServers": {
       "harness-builder": {
         "command": "node",
         "args": ["/path/to/dist/index.js"]
       }
     }
   }
   ```

## ❓ 자주 묻는 질문

**Q: 이 플러그인을 바로 사용할 수 있나요?**
A: 네! 다만 아직 NPM에 퍼블리시되지 않았으므로, 현재는 로컬에서 빌드하거나 `npx`로 직접 실행 가능하도록 설정하면 됩니다.

**Q: 다른 프레임워크(Playwright, Cypress 등)도 지원하나요?**
A: 현재는 Jest, Mocha, Vitest만 지원합니다만, COMPLETE_GUIDE.md의 "확장" 섹션을 참고해 쉽게 추가할 수 있습니다.

**Q: Python 프로젝트에 사용할 수 있나요?**
A: 현재는 TypeScript/JavaScript용입니다. Python 지원은 "확장 및 커스터마이징" 섹션의 새로운 언어 추가 방법을 참고하면 됩니다.

**Q: Claude Code에 연결했는데 도구가 안 보여요.**
A: SETUP.md의 "문제 해결" 섹션을 참고해주세요. 대부분 설정 파일 포맷 문제입니다.
