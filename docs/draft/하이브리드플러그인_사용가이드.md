## 🚀 3가지 사용 방식

### 방식 1️⃣: Claude Code 플러그인으로만 사용

```
/plugin install github:yourusername/harness-hybrid-plugin

사용:
/harness-init my-project
/test-case LoginTest "설명"
/integration-tests
/performance-test api 100
/harness-config
```

**장점:** 가장 간단, 슬래시 커맨드, 설정 UI

### 방식 2️⃣: MCP 서버로만 사용

```
설정 파일:
{
  "mcpServers": {
    "harness-builder": {
      "command": "node",
      "args": ["dist/hybrid-server.js"]
    }
  }
}

사용:
도구 선택 → init_harness → 실행
```

**장점:** 유연성, 다양한 LLM 도구 지원

### 방식 3️⃣: 🏆 플러그인 + MCP 함께 사용 (권장!)

```
플러그인 설치:
/plugin install github:yourusername/harness-hybrid-plugin

MCP도 추가:
설정 파일에 MCP 서버 등록

사용:
- 빠른 작업: /harness-init
- 복잡한 작업: MCP 도구 사용
- 설정: 설정 UI에서 변경
```

**장점:** 모든 기능 + 최대 유연성!

## 🔧 내부 구조

```
hybrid-server.ts
    ↓
    ├── PART 1: Claude Code 플러그인
    │   ├── HarnessBuilderPlugin 클래스
    │   ├── activate() - 플러그인 활성화
    │   └── registerSlashCommands() - 5가지 커맨드
    │
    ├── PART 2: MCP 서버
    │   ├── startMCPServer() - MCP 서버 시작
    │   ├── list_tools - 도구 목록
    │   └── call_tool - 도구 실행
    │
    └── PART 3: 하이브리드 실행
        ├── getExecutionMode() - 모드 감지
        └── main() - 자동 선택 실행
```

## 🎯 자동 모드 감지

플러그인은 **자동으로 실행 환경을 감지**합니다:

```typescript
// Claude Code 플러그인 환경 감지
if (typeof window !== "undefined" && "claudeCode" in window) {
  → 플러그인 모드로 실행
}

// MCP 환경 감지
if (process.env.MCP_MODE === "true") {
  → MCP 모드로 실행
}

// 기본값 (하이브리드)
→ 플러그인 + MCP 모두 지원
```
